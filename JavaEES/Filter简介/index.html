    Filter简介

    JavaEES Web java Filter
    创建时间:2019-09-25 15:40
    阅读:
    1. 1. 创建并配置Filter

    Filter顾名思义就是一种过滤器，是对Servlet的加强，主要负责对用户的请求信息和服务器返回信息的预处理和过滤，是一个典型的处理链。有如下几个用处:

    • 在HttpServletRequest到达Servlet之前，拦截HttpServletRequest请求
    • 根据需要检查和修改HttpServletRequest的信息
    • 在HttpServletResponse返回客户端之间，拦截HttpServletResponse
    • 根据需要检查和修改HttpServletResponse的信息

    Filter主要由如下几类：

    • 用户授权Filter：负责检查用户请求，过滤非法请求
    • 日志Filter：记录特殊的请求信息
    • 解码Filter：对非标准编码的请求进行解码
    • SXLT Filter：修改XML请求内容
    • 拦截Filter：拦截用户请求和服务器响应。

    那么如何使用Filter呢？一般需要以下两个步骤：

    1. 创建Filter处理类
    2. 采用web.xml方式或注解的方式配置Filter

    1. 创建并配置Filter

    创建Filter必须实现javax.servlet.Filter接口，该接口的定义如下：

    public interface Filter {

    /** 
     * Called by the web container to indicate to a filter that it is
     * being placed into service.
     *
     * <p>The servlet container calls the init
     * method exactly once after instantiating the filter. The init
     * method must complete successfully before the filter is asked to do any
     * filtering work.
     * 
     * <p>The web container cannot place the filter into service if the init
     * method either
     * <ol>
     * <li>Throws a ServletException
     * <li>Does not return within a time period defined by the web container
     * </ol>
     */
    public void init(FilterConfig filterConfig) throws ServletException;


    /**
     * The <code>doFilter</code> method of the Filter is called by the
     * container each time a request/response pair is passed through the
     * chain due to a client request for a resource at the end of the chain.
     * The FilterChain passed in to this method allows the Filter to pass
     * on the request and response to the next entity in the chain.
     *
     * <p>A typical implementation of this method would follow the following
     * pattern:
     * <ol>
     * <li>Examine the request
     * <li>Optionally wrap the request object with a custom implementation to
     * filter content or headers for input filtering
     * <li>Optionally wrap the response object with a custom implementation to
     * filter content or headers for output filtering
     * <li>
     * <ul>
     * <li><strong>Either</strong> invoke the next entity in the chain
     * using the FilterChain object
     * (<code>chain.doFilter()</code>),
     * <li><strong>or</strong> not pass on the request/response pair to
     * the next entity in the filter chain to
     * block the request processing
     * </ul>
     * <li>Directly set headers on the response after invocation of the
     * next entity in the filter chain.
     * </ol>
     */
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain)
            throws IOException, ServletException;


    /**
     * Called by the web container to indicate to a filter that it is being
     * taken out of service.
     *
     * <p>This method is only called once all threads within the filter's
     * doFilter method have exited or after a timeout period has passed.
     * After the web container calls this method, it will not call the
     * doFilter method again on this instance of the filter.
     *
     * <p>This method gives the filter an opportunity to clean up any
     * resources that are being held (for example, memory, file handles,
     * threads) and make sure that any persistent state is synchronized
     * with the filter's current state in memory.
     */
    public void destroy();
}

    方法的注释已经对每个方法的功能进行了详细的描述。

    接下来，我们对Servlet简介一章里面提到的用户登录验证功能进行拆分，定义一个用户授权的Filter，该Filter对用户的请求进行过滤，验证用户是否已经登录，如果用户已经登录，则直接进去请求的页面，否则跳转到登录界面。下面给出了具体的代码：

    package io.github.abbottkilig.filter;

import java.io.IOException;

import javax.servlet.Filter;
import javax.servlet.FilterChain;
import javax.servlet.FilterConfig;
import javax.servlet.ServletException;
import javax.servlet.ServletRequest;
import javax.servlet.ServletResponse;
import javax.servlet.annotation.WebFilter;
import javax.servlet.annotation.WebInitParam;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpSession;

import com.mysql.cj.Session;

@WebFilter(filterName = "authority", urlPatterns = {"/*"},
            initParams = {
                    @WebInitParam(name = "loginPage", value = "/login.jsp"),
                    @WebInitParam(name = "welcomePage", value = "/welcome.jsp")
            }
        )
public class AuthorityFilter implements Filter {

    private FilterConfig config;

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        this.config = filterConfig;
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
             HttpServletRequest hsr =  (HttpServletRequest)request;
             HttpSession session = hsr.getSession(true);
             if (session.getAttribute("name") == null &&!hsr.getRequestURI().equals(config.getInitParameter("loginPage")) {
                 request.setAttribute("error", "please login!");
                hsr.getRequestDispatcher(config.getInitParameter("loginPage")).forward(request, response);
            }else {
                chain.doFilter(request, response);
                System.out.println("Request is granted permission");
            }
    }

    @Override
    public void destroy() {
        this.config = null;
    }

}

    代码中采用了注解的方式配置了Filter,其配置与Servlet相当类似，都需要配置以下两个部分：

    • Filter的名称
    • 配置Filter拦截的URL模型。注意此处urlPatterns为字符串数组，表示可以运用到多个URL模型上

    同样的，我们给出AuthorityFilter在web.xml中的配置

    <filter>
    <filter-name>authority</filter-name>
    <filter-class>io.github.abbottkilig.filter.AuthorityFilter</filter-class>
    <init-param>
        <param-name>loginPage</param-name>
        <param-value>/login.jsp</param-value>
    </init-param>

    <init-param>
        <param-name>welcomePage</param-name>
        <param-value>/welcome.jsp</param-value>
    </init-param>
</filter>

<filter-mapping>
    <filter-name>authority</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

    此时，当我们访问http://localhost/demo/welcome.jsp或demo应用下的任意一个界面，都会跳转到如图1-1所示的login.jsp页面，要求用户登陆。

    既然之前提到的Servlet可以做这样的逻辑，为什么还要将验证逻辑提取出来放在Filter里面呢？原因有以下几点：

    • Servlet通常只能匹配一个URL，而Filter可以同时拦截多个请求的URL。因此，如果需要多Web应用的多个请求都做登录验证，在每个Servlet里面写验证功能将造成大量的代码冗余，将多个Servlet里面共同的逻辑提取出来形成Filter，将能够极大的提高代码的复用性。

    • 对于一个Web应用来说，许多Servlet都需要一些通用的操作，如权限控制、日志记录等等，这些在Servlet的service()方法中的代码都是相同的，可以考虑将这样的通用处理逻辑提取到Filter中完成。
