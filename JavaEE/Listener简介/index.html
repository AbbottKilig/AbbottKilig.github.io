    Listener简介

    JavaEE Web java Listener
    创建时间:2019-09-25 19:38

    监听器用来监听Web应用内的内部事件，如Web应用启动、停止，用户session会话开始、接受，用户请求到达等。对于开发者而言，这些信息可以用来判断程序的运行状态、负载等，将是非常重要的信息。

    但是，要记录这样的信息，一种方式是在Web应用相关的每个函数内都加上响应的日志信息。如果后日要更改日志格式等，就必须修改大量的Web应用的源代码，造成维护成本很大。

    因此，为了解决这样的问题，Listener孕育而生，通过单独的Listener类，监听Web应用的状态。

    使用Listener需要两个步骤：

    • 定义Listener实现类
    • 使用注解或web.xml的方式配置listener

    1. 编写Listener类

    对于Web应用而言，不同的Web事件对应的监听器各不相同，常用的Web时间监听器接口有：

    • ServletContextListener:监听Web应用的启动和关闭
    • ServletContextAttributeListener：监听ServletContext范围内属性的改变
    • ServletRequestListener：监听用户请求
    • ServletRequestAttributeListener：监听ServletRequest范围内属性的改变
    • HttpSessionListener：监听session的开始和结束
    • HttpSessionAttributeListener：监听HttpSession范围内属性的改变

    接下来，分别介绍几个监听器的开发和使用。

    1.1 ServletContextListener开发

    ServletContextListener用去监听Web应用的启动和关闭，该Listener必须实现ServletContextListener接口，且接口包含如下两个方法：

    public interface ServletContextListener extends EventListener {

    /**
     * Receives notification that the web application initialization
     * process is starting.
     *
     * <p>All ServletContextListeners are notified of context
     * initialization before any filters or servlets in the web
     * application are initialized.
     *
     * @param sce the ServletContextEvent containing the ServletContext
     * that is being initialized
     */
    public void contextInitialized(ServletContextEvent sce);

    /**
     * Receives notification that the ServletContext is about to be
     * shut down.
     *
     * <p>All servlets and filters will have been destroyed before any
     * ServletContextListeners are notified of context
     * destruction.
     *
     * @param sce the ServletContextEvent containing the ServletContext
     * that is being destroyed
     */
    public void contextDestroyed(ServletContextEvent sce);
}

    接下来，创建一个获取数据库连接的Listener,该Listener在应用启动时获取数据库的连接，并将取得的链接设置成application范围内的属性。在demo工程io.github.abbottkilig.listener包下新建getConnListener类，拷贝以下代码：

    package io.github.abbottkilig.listener;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

import javax.servlet.ServletContext;
import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebInitParam;
import javax.servlet.annotation.WebListener;


@WebListener()
public class GetConnListener implements ServletContextListener {

    @Override
    public void contextInitialized(ServletContextEvent sce) {
        ServletContext servletContext = sce.getServletContext();
        String driver = servletContext.getInitParameter("driver");
        String url = servletContext.getInitParameter("url");
        String username = servletContext.getInitParameter("username");
        String password = servletContext.getInitParameter("password");
        try {
            Class.forName(driver);
            Connection conn = DriverManager.getConnection(url, username, password);
            servletContext.setAttribute("conn", conn);
        } catch (Exception e) {
            System.out.println("database cann't be connected");
            System.out.println(e.getMessage());
        }

    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {
        ServletContext servletContext = sce.getServletContext();
        Connection conn = (Connection)servletContext.getAttribute("conn");
        try {

        if (conn != null || !conn.isClosed()) {
            conn.close();
            sce.getServletContext().removeAttribute("conn");
        }
        }
        catch (SQLException e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
    }

}

    从代码中可以看到，getConnListener实现了ServletContextListener接口规定的两个方法contextnitialized()和contextDestroyed()contextnitialized()实现在Web应用启动的时候启动数据库的连接，contextDestroyed()实现在关闭Web应用的时候关闭数据库的链接，实现为整个Web应用提供数据库连接服务。

    1.2 ServletContextAttributeListener开发

    ServletContextAttributeListener用于监听ServletContext范围内的属性变化，查看该接口源码，需要实现其三个方法：

    public interface ServletContextAttributeListener extends EventListener {

    /**
     * Receives notification that an attribute has been added to the
     * ServletContext.
     *
     * @param event the ServletContextAttributeEvent containing the
     * ServletContext to which the attribute was added, along with the
     * attribute name and value
     */
    public void attributeAdded(ServletContextAttributeEvent event);

    /**
     * Receives notification that an attribute has been removed
     * from the ServletContext.
     *
     * @param event the ServletContextAttributeEvent containing the
     * ServletContext from which the attribute was removed, along with
     * the attribute name and value
     */
    public void attributeRemoved(ServletContextAttributeEvent event);

    /*
     * Receives notification that an attribute has been replaced
     * in the ServletContext.
     *
     * @param event the ServletContextAttributeEvent containing the
     * ServletContext in which the attribute was replaced, along with
     * the attribute name and its old value
     */
    public void attributeReplaced(ServletContextAttributeEvent event);
}

    下面给出监听ServletContext范围内属性变化的Listener。在的demo工程包io.github.abbottkilig.listener下新建CustomerServletAttributeListener.java，拷贝以下代码

    package io.github.abbottkilig.listener;

import javax.servlet.ServletContextAttributeEvent;
import javax.servlet.ServletContextAttributeListener;
import javax.servlet.annotation.WebListener;

@WebListener
public class CustomerServletAttributeListener implements ServletContextAttributeListener{

    @Override
    public void attributeAdded(ServletContextAttributeEvent event) {
        String name = event.getName();
        Object value = event.getValue();
        System.out.println(event.getServletContext().getServletContextName() + "add a new attribute (name=" + name + ",value=" + value + ")");
    }

    @Override
    public void attributeRemoved(ServletContextAttributeEvent event) {
        String name = event.getName();
        Object value = event.getValue();
        System.out.println(event.getServletContext().getServletContextName() + "removes a attribute (name=" + name + ",value=" + value + ")");
    }

    @Override
    public void attributeReplaced(ServletContextAttributeEvent event) {

        String name = event.getName();
        Object value = event.getValue();
        System.out.println(event.getServletContext().getServletContextName() + "changes a attribue (name=" + name + ",value=" + value + ")");
    }

}

    该监听器通过三个方法对Web应用内的属性添加、删除、修改进行了监听。当放生对应的事件时，该监听器将被触发。

    注意：这里注解依旧使用的是@WebListener

    1.3 ServletRequestListener和ServletRequestAttributeListener类

    和之前小节的内容比较相似，ServletRequestListener类用于监听用户请求的到达，ServletRequestAttributeListener则用于监听ServletRequest范围内属性的变动。

    ServletRequestListener接口规定了以下方法：

    public interface ServletRequestListener extends EventListener {

    /**
     * Receives notification that a ServletRequest is about to go out
     * of scope of the web application.
     *
     * @param sre the ServletRequestEvent containing the ServletRequest
     * and the ServletContext representing the web application
     */
    public void requestDestroyed(ServletRequestEvent sre);

    /**
     * Receives notification that a ServletRequest is about to come
     * into scope of the web application.
     *
     * @param sre the ServletRequestEvent containing the ServletRequest
     * and the ServletContext representing the web application
     */
    public void requestInitialized(ServletRequestEvent sre);
}

    ServletRequestAttributeListener接口规定了以下方法：

    public interface ServletContextAttributeListener extends EventListener {

    /**
     * Receives notification that an attribute has been added to the
     * ServletContext.
     *
     * @param event the ServletContextAttributeEvent containing the
     * ServletContext to which the attribute was added, along with the
     * attribute name and value
     */
    public void attributeAdded(ServletContextAttributeEvent event);

    /**
     * Receives notification that an attribute has been removed
     * from the ServletContext.
     *
     * @param event the ServletContextAttributeEvent containing the
     * ServletContext from which the attribute was removed, along with
     * the attribute name and value
     */
    public void attributeRemoved(ServletContextAttributeEvent event);

    /*
     * Receives notification that an attribute has been replaced
     * in the ServletContext.
     *
     * @param event the ServletContextAttributeEvent containing the
     * ServletContext in which the attribute was replaced, along with
     * the attribute name and its old value
     */
    public void attributeReplaced(ServletContextAttributeEvent event);
}

    在开发过程中，为了简化，可以用一个类来实现多个监听器的接口，以实现对多种实践的监听。

    下面给出一个具体的的实例。

    在的demo工程包io.github.abbottkilig.listener下新建RequestListener.java，拷贝以下代码

    package io.github.abbottkilig.listener;

import javax.servlet.ServletContextAttributeEvent;
import javax.servlet.ServletContextAttributeListener;
import javax.servlet.ServletRequestAttributeEvent;
import javax.servlet.ServletRequestAttributeListener;
import javax.servlet.ServletRequestEvent;
import javax.servlet.ServletRequestListener;
import javax.servlet.http.HttpServletRequest;
@WebListener
public class RequestListener implements ServletRequestListener, ServletRequestAttributeListener {



    @Override
    public void requestDestroyed(ServletRequestEvent sre) {

        HttpServletRequest req = (HttpServletRequest)sre.getServletRequest();
        System.out.println(req.getRequestURI() + "was requested by " + req.getRemoteAddr() + ":" + req.getRemotePort());
    }

    @Override
    public void requestInitialized(ServletRequestEvent sre) {
        HttpServletRequest req = (HttpServletRequest)sre.getServletRequest();
        System.out.println(req.getRemoteAddr() + ":" + req.getRemotePort() + " is requesting "+ req.getRequestURI());
    }

    @Override
    public void attributeAdded(ServletRequestAttributeEvent srae) {
        String name = srae.getName();
        Object value = srae.getValue();
        System.out.println(srae.getServletRequest() + "add a attribute (name=" + name + ",value=" + value + ")");
    }

    @Override
    public void attributeRemoved(ServletRequestAttributeEvent srae) {
        String name = srae.getName();
        Object value = srae.getValue();
        System.out.println(srae.getServletRequest() + "removes a attribute (name=" + name + ",value=" + value + ")");

    }

    @Override
    public void attributeReplaced(ServletRequestAttributeEvent srae) {
        String name = srae.getName();
        Object value = srae.getValue();
        System.out.println(srae.getServletRequest() + "changes a attribue (name=" + name + ",value=" + value + ")");

    }



}

    1.4 HttpSessionListener和HtppSessionAttributeListener

    和之前小节的内容比较相似，HttpSessionListenerr类用于session的创建和销毁，HtppSessionAttributeListener则用于监听Session范围内属性的变动。

    ServletRequestListener接口规定了以下方法：

    public interface HttpSessionListener extends EventListener {

    /** 
     * Receives notification that a session has been created.
     *
     * @param se the HttpSessionEvent containing the session
     */
    public void sessionCreated(HttpSessionEvent se);

    /** 
     * Receives notification that a session is about to be invalidated.
     *
     * @param se the HttpSessionEvent containing the session
     */
    public void sessionDestroyed(HttpSessionEvent se);

}

    ServletRequestAttributeListener接口规定了以下方法：

    public interface HttpSessionAttributeListener extends EventListener {

    /**
     * Receives notification that an attribute has been added to a
     * session.
     *
     * @param event the HttpSessionBindingEvent containing the session
     * and the name and value of the attribute that was added
     */
    public void attributeAdded(HttpSessionBindingEvent event);

    /**
     * Receives notification that an attribute has been removed from a
     * session.
     *
     * @param event the HttpSessionBindingEvent containing the session
     * and the name and value of the attribute that was removed
     */
    public void attributeRemoved(HttpSessionBindingEvent event);

    /**
     * Receives notification that an attribute has been replaced in a
     * session.
     *
     * @param event the HttpSessionBindingEvent containing the session
     * and the name and (old) value of the attribute that was replaced
     */
    public void attributeReplaced(HttpSessionBindingEvent event);

}

    下面给出一个具体的的实例。

    在的demo工程包io.github.abbottkilig.listener下新建SessionListener.java，拷贝以下代码

    package io.github.abbottkilig.listener;

import javax.servlet.annotation.WebListener;
import javax.servlet.http.HttpSessionActivationListener;
import javax.servlet.http.HttpSessionAttributeListener;
import javax.servlet.http.HttpSessionBindingEvent;
import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;

@WebListener
public class SessionListener
        implements HttpSessionListener, HttpSessionAttributeListener {


    @Override
    public void attributeAdded(HttpSessionBindingEvent event) {
        String name = event.getName();
        String value = (String)event.getValue();
        System.out.println("session adds a new attribute(name=" +name + "value=" + value+")");
    }

    @Override
    public void attributeRemoved(HttpSessionBindingEvent event) {
        String name = event.getName();
        String value = (String)event.getValue();
        System.out.println("session removes a  attribute(name=" +name + "value=" + value+")");
    }

    @Override
    public void attributeReplaced(HttpSessionBindingEvent event) {
        String name = event.getName();
        String value = (String)event.getValue();
        System.out.println("session changes a  attribute(name=" +name + "value=" + value+")");
    }

    @Override
    public void sessionCreated(HttpSessionEvent se) {
        String id = se.getSession().getId();
        String user = (String)se.getSession().getAttribute("user");

        System.out.println("session " + id + " is created between " + se.getSession().getServletContext() + "and " + user);
    }

    @Override
    public void sessionDestroyed(HttpSessionEvent se) {
        String id = se.getSession().getId();
        String user = (String)se.getSession().getAttribute("user");

        System.out.println("session " + id + " is closed between " + se.getSession().getServletContext() + "and " + user);
    }

}

    2. Listener配置

    配置监听器和配置Servlet、Filter一样有两种方式：

    • 注解配置

      • 注解配置使用@WebListener注解即可向Web应用注册该监听器

      • @WebListener的源码如下，说明其不像Servlet、Filter需要配置其他内容

        @Target({ElementType.TYPE})
        @Retention(RetentionPolicy.RUNTIME)
        @Documented
        public @interface WebListener {

        /**
 * Description of the listener
 */
String value() default "";

        }

    • web.xml配置

      • web.xml中使用<listener>节点进行配置
      • 在<listener>节点中指定Listener的实现类

    下面给出Listener类的web.xml配置模板，其中*表示具体的需要添加的监听器类

    <listener>
    <listener-class>io.github.abbottkilig.listener.*</listener-class>
</listener>
