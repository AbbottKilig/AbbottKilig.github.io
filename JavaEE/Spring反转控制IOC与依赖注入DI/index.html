    Spring反转控制IOC与依赖注入DI

    JavaEE Spring Web
    创建时间:2019-09-27 10:26
    阅读:
    1. 1.1 Spring IOC和Bean简介
      1. 1.2 spring容器概述
        1. 1.2.1 元数据配置
        2. 1.2.2 容器实例化

    Spring框架中最重要的就是控制反转IOC，本节将对IOC进行详细的介绍。

    1.1 Spring IOC和Bean简介

    本节介绍了Spring框架中反转控制IOC的实现。IOC也称为依赖注入DI。在此过程中，Spring容器在创建Bean对象时，仅通过构造函数的参数，get/set方法的参数来注入其依赖项。

    Spring框架的org.springframework.beans和org.springframework.context包是IOC实现的基础。org.springframework.beans包中定义的BeanFactroty接口提供了一种可以管理任何对象的高级配置机制。ApplicationContext继承了BeanFactory，增加了：

    • AOP集成
    • 消息处理
    • 事件发布

    简而言之，BeanFactory提供了基本的配置框架和功能。ApplicationContext是BeanFactory的子类，但增加了更多企业实用的功能。

    Spring中，由Spring控制的构建应用程序的主干对象称为bean。bean通过spring IOC组装、实例化、管理。容器的配置元数据反映了bean及其之间的依赖关系。

    1.2 spring容器概述

    org.springframework.context.ApplicationContext接口表现了spring IOC的功能：负责bean的实例化、组装、配置。容器通过以xml或Java注解的方式提供的元数据来时时间这样的功能。

    在实际应用中，通常创建org.springframework.context.ApplicationContext接口的具体实现类ClassPathXmlApplicationContext或FileSystemXmlApplicationContext来实现读取xml文件元数据的配置。

    下图显示了Spring的工作原理的高级视图。应用程序类和配置元数据结合在一起，在ApplicationContext创建成功并初始化之后，基于配置的应用程序能够启动正常服务。

    1.2.1 元数据配置

    基于历史传统，Spring元数据的配置也采用了简单直观的xml格式。但是，基于xml的元数据不是配置元数据的唯一形式，Spring IOC也提供了如下的两种方式：

    • 基于注解的配置：spring 2.5引入的基于注解的配置方式
    • 基于Java的配置：从spring 3.0开始，JavaConfig项目提供的很多功能成为了spring framework的一部分。因此，也可以采用的java的配置。

    基于xml的配置中，bean的配置由<beans>顶级元素的<bean>指定。Java配置通常在@Configuration类中使用带@Bean注解的方法。

    bean定义了组成应用程序的实际对象，如服务层对象，数据访问对象，表示层对象，基础结构对象等。通常，不会在容器中配置细粒度的域对象，因为创建和加载域对象通常是DAO和业务逻辑的职责。

    下面给出xml配置元数据的基本结构。

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="..." class="...">   
        <!-- collaborators and configuration for this bean go here -->
    </bean>

    <bean id="..." class="...">
        <!-- collaborators and configuration for this bean go here -->
    </bean>

    <!-- more bean definitions go here -->

</beans>

    id属性用于唯一标示单个bean
    class属性定义bean的类型，应采用全类限定名。

    1.2.2 容器实例化

    提供给ApplicationContext构造函数的一个或多个位置路径可以从各种外部资源（例如本地文件系统，Java等）加载配置元数据CLASSPATH。

    接下来，作为具体示例，我们将依靠spring IOC对demo项目进行进一步改造，建立数据库查询服务。

    首先，通过ApplicationContext加载元数据配置的xml文件。

    ApplicationContext context = new ClassPathXmlApplicationContext("services.xml","daos.xml");

    然后，给出services.xml文件的配置

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="daoservice" class="io.github.abbottkilig.services.DAOService">   
        <property name="dbu" ref="dbutil">
        </property>
    </bean>
</beans>

    在xml文件中定义了一个类型为io.github.abbottkilig.services.DAOService的bean，该bean的代码如下：

    package io.github.abbottkilig.services;

import io.github.abbottkilig.utils.DBUtil;

public class DAOService {


    private DBUtil dbu;

    public DAOService(DBUtil dbu) {
        super();
        this.dbu = dbu;
    }

    public DAOService() {
    }

    public DBUtil getDbu() {
        return dbu;
    }

    public void setDbu(DBUtil dbu) {
        this.dbu = dbu;
    }



}

    接下来，<property>元素表明采用将采用bean的set方法来注入依赖项，而ref则指明了依赖的bean的id。

    接下来，我们给出daos.xml文件的配置：

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="dbutil" class="io.github.abbottkilig.utils.DBUtil">   
        <constructor-arg name="driver" value="com.mysql.cj.jdbc.Driver"></constructor-arg>
        <constructor-arg name="url" value="jdbc:mysql://localhost:3306/wechat?useSSL=false&amp;allowPublicKeyRetrieval=true&amp;serverTimezone=UTC"></constructor-arg>
        <constructor-arg name="username" value="root"></constructor-arg>
        <constructor-arg name="password" value="71939224@qq.com"></constructor-arg>
    </bean>
</beans>

    xml配置文件中，定义了一个类型为io.github.abbottkilig.utils.DBUtil的bean，并通过<constructor-arg>元素通过构造器注入了元素值。

    组成基于XML的配置元数据

    通过前面的配置示例看到，使bean配置跨越了多个xml文件。这样做的原因是：可以通过单独的xml配置文件来代表应用体系中的逻辑层或模块，达到分模块开发维护的目的。
