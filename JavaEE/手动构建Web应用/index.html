    手动构建Web应用

    JavaEE Web
    创建时间:2019-09-21 19:57
    阅读:
    1. 1. 创建Web应用目录结构
    2. 2. 配置web.xml描述符

    1. 创建Web应用目录结构

    JSP、Servlet、Listener和Filter等都必须运行在Web应用中，虽然可以借助Eclipse、IDEA等快速构建Web应用生成对应的目录结构，但是对于开发者而言，深刻的理解Web应用的目录结构弥足关键，因此需要学习如何手动构建一个Web应用。

    坚持使用最原始的工具进行学习，从基本功练起，虽然一开始的过程有些痛苦，但到最后能让开发者对技术的每个细节都有更准备的把握，真正成为一名优秀的程序员。

    接下来，将带领大家徒手构建一个Web应用，需按如下的步骤进行：

    1. 在任意目录下新建一个文件夹，本文采用在D:/project下新建一个demo文件夹来构建一个Web应用

    2. 在demo文件下新建WEB-INF文件夹

    3. 在WEB-INF文件夹下新建web.xml文件，并将如下的内容拷贝到web.xml中。

    <?xml version="1.0" encoding="UTF-8"?>

<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
                      http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
  version="4.0"
  metadata-complete="true">

</web-app>

    1. 在WEB-INF文件下新建classes和lib两个文件夹，这两个文件夹都用来保存Web应用所需要的Java类文件，区别是classes保存Web应用生成的单个class文件，lib文件夹保存的是Web应用依赖的jar包

    2. 在demo目录下新建hello.jsp文件，复制以下内容

      <%@ page language="java" contentType="text/html; charset=utf-8"
pageEncoding="utf-8"%>
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>hello Web</title>
</head>
<body>
<h1>Hello World</h1>
</body>
</html>

    3. 此时已经构建完成了一个基本的空的Web应用。将该应用复制到Tomcat的webapps目录下，该Web应用便可以自动部署在Tomcat上了。

    4. 启动Tomcat服务器, 浏览器地址输出http://127.0.0.1/demo/hello.jsp,即可以看到如图1-1的显示的页面，说明Web应用成功构建并部署到Tomcat。

    2. 配置web.xml描述符

    在上一节中，说到构建Web应用需要在WEB-INF目录下创建web.xml。事实上，web.xml文件被称为Java Web应用的配置描述符，在Servlet2.5规范之前，每个Java Web应用都必须包含一个存放在WEB-INF路径下的web.xml文件，而且绝大部分组件都是通过此文件来进行管理配置。

    从Servlet3开始，web.xml文件不再是Java Web应用必须的文件，Servelet3可以通过注解的方式来配置管理Web组件。

    对于Java Web应用，WEB-INF是一个特殊的文件夹，Web容器会包含该文件夹下的内容，即使Tomcat在自身的web.xml文件中打开的listings节点，客户端浏览器也无法访问该路径下的任何内容。

    通过对web.xml文件的修改，通常可以配置Java Web应用的如下内容：

    • 配置JSP
    • 配置管理Servlet
    • 配置管理Listener
    • 配置管理Filter
    • 配置标签库
    • 配置JSP属性

    而且，web.xml文件还可以负责配置管理如下的内容：

    • 配置管理JAAS授权认证
    • 配置和管理资源引用
    • Web应用首页

    在上一节中，创建了一个包含如下内容的默认web.xml文件。该文件的根元素是<web-app>,其中属性metadata-complete是属于Servlet3规范中新增的属性。该属性为true时，Web应用不会加载注解配置的Web组件（如Servlet、Filter、Listener等）

    <?xml version="1.0" encoding="UTF-8"?>

    <web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
                          http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
      version="4.0"
      metadata-complete="true">

</web-app>

    在web.xml的web-app节点下，可以创建配置Web应用首页的节点元素welcome-file-list,该元素可以包含多个welcome-file子元素，其中每个welcom-file子元素配置了一个Web首页，例如，如下的配置片段：

    <welcome-file-list>
    <welcome-file>index.html</welcome-file>
    <welcome-file>index.xhtml</welcome-file>
    <welcome-file>index.htm</welcome-file>
    <welcome-file>index.jsp</welcome-file>
</welcome-file-list>

    该片段依次配置了Web应用的首页是index.html，index.xhtml，index.htm，index.jsp。当浏览者访问该Web应用时，系统将按照此顺序给浏览者呈现首页。即当index.html不存在时，展示index.xhtml；index.xhtml不存在时，展示index.htm；依次类推。

    对上一节提到的提到的Web应用demo的web.xml文件进行修改，添加welcome-file-list，形成最终的web.xml文件如下

    <?xml version="1.0" encoding="UTF-8"?>

<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
                      http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
  version="4.0"
  metadata-complete="true">

  <welcome-file-list>
        <welcome-file>index.html</welcome-file>
        <welcome-file>index.xhtml</welcome-file>
        <welcome-file>index.htm</welcome-file>
        <welcome-file>hello.jsp</welcome-file>
    </welcome-file-list>

</web-app>

    此时，通过浏览器直接访问网址http://localhost:8080/demo，既可以看到图1-1中显示的内容。

    当在demo目录下新建包含如下内容的index.html时，重新访问http://localhost:8080/demo，则会看到如图1-2所示的内容。

    <!DOCTYPE html>
<html>
<head>
    <title>index.html</title>
</head>
<body>
<h1>This is index.html</h1>
</body>
</html>
